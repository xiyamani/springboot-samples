# 知乎专题列表 — 开发文档

本文档说明 `zhihu-vue` 项目中专题列表页的实现方式：依赖安装、接口约定、数据映射、开发与构建、样式与知乎专题广场的对应关系，以及常见问题。

---

## 1. 项目简介

前端使用 **Vue 3 + TypeScript + Vite**，通过 **Axios** 请求后端专题接口，使用 **Element Plus** 提供输入框、按钮、分页、骨架屏与空状态等组件。列表布局与视觉参考 [知乎专题广场 / 全部专题](https://www.zhihu.com/special/all)：浅灰背景、白底卡片、左侧封面图、标题与元信息（更新时间、浏览量）、右侧操作按钮风格接近「关注专题」。

默认请求与下列接口等价（后端需在本机 `8080` 端口提供）：

`GET http://localhost:8080/api/v1/special/selectByTitle?title=安全&pageNum=1&pageSize=10`

开发环境下，浏览器只访问**同源**的 `/api/...`，由 Vite 将请求**代理**到 `http://localhost:8080`，避免跨域。

## 3. 目录结构说明

本节用「树形索引 + 分目录说明 + 调用关系」三部分，说明代码放在哪里、各文件职责、以及专题功能在运行时如何串联。

```

│   ├── router/
│   │   └── index.ts            # 路由表：/special、/special/:id
│   ├── types/
│   │   └── special.ts          # SpecialItem、SpecialPageResult 等 TS 类型

│   ├── vite-env.d.ts           # 环境变量类型（如 VITE_API_BASE）
│   └── style.css               # Vite 模板遗留的全局样式（当前 main 未引入，按需使用）
```

`npm run build` 生成的 **`dist/`** 为发布用静态资源，勿手改；线上一般由 CI 构建或由脚本上传 `dist`。

### 3.2 分目录职责（该往哪儿改）

| 路径 | 职责 | 常见改动场景 |
|------|------|----------------|
| **`src/views/`** | 页面：拉数据、加载态、布局、组合子组件 | 列表/详情交互、筛选项、分页策略 |
| **`src/components/`** | 可复用展示或小块交互 | 卡片样式、拆更小子组件 |
| **`src/api/special.ts`** | 声明 URL、query，调用 `httpGet` | 后端改路径/参数、新增接口方法 |
| **`src/api/specialNormalize.ts`** | 兼容多种响应包装并映射 DTO | 字段名变化、`data` 嵌套方式变化 |
| **`src/utils/http.ts`** | 全局请求：baseURL、超时；导出 `httpGet` | 网关前缀、Token 拦截、统一错误码 |
| **`src/utils/httpError.ts`** | 异常 → 一行可读文案 | 与 `ElMessage` 提示口径统一 |
| **`src/router/`** | URL 与组件对应、重定向 | 增加新页面路由 |
| **`src/types/special.ts`** | 前端类型契约 | 与后端 DTO 同步增删字段 |
| **`vite.config.ts`** | 开发服务器、代理、别名 | 后端端口非 8080、多环境代理 |

### 3.3 专题列表到接口的调用链（数据流）

排查「有没有请求后端、数据在哪一步丢了」时可按下面顺序核对：

1. 浏览器进入 **`/special`** → **`router/index.ts`** 加载 **`SpecialListView.vue`**。
2. 列表页调用 **`fetchSpecialByTitle`**（**`api/special.ts`**）→ **`httpGet('/v1/special/selectByTitle', { params })`**（**`utils/http.ts`**）。
3. 实际 URL 为 **`baseURL` + 上述 path**；开发环境常将 **`baseURL` 设为 `/api`**，由 **`vite.config.ts`** 代理到 **`http://localhost:8080`**。
4. 响应体交给 **`specialNormalize.ts`**：`pickRows` / `pickTotal` 取列表与总数，**`normalizeItem`** 得到 **`SpecialItem[]`**。
5. 列表用 **`SpecialCard.vue`** 渲染；点击进入 **`/special/:id`** → **`SpecialDetailView.vue`** 调用 **`fetchSpecialById`**，同样经过 **`httpGet`** → **`pickOneRecord`** → **`normalizeItem`**。

若接口 200 但列表仍空，多数在 **`specialNormalize.ts`** 与真实 JSON 结构不一致；若无请求或 404，先查 **`http.ts` 的 baseURL** 与 **Vite 代理 target**。

## 4. 环境要求

- **Node.js**：建议当前 LTS（如 20+），需能运行 `npm`。
- **后端**：在 `http://localhost:8080` 提供  
  - `GET /api/v1/special/selectByTitle`（列表）  
  - `GET /api/v1/special/selectById?id=`（详情，路径可与后端约定后改 `src/api/special.ts` 一处）

若后端地址或路径不同，请修改 `vite.config.ts` 中的 `server.proxy` 以及 `src/utils/http.ts` 中的 `baseURL` 逻辑。

## 5. 安装与运行

### 5.1 安装依赖

在项目根目录执行：

```bash
npm install
```

已声明的主要依赖包括：`vue`、`vue-router`、`axios`、`element-plus`。

### 5.2 启动开发服务器

```bash
npm run dev
```

浏览器访问终端输出的本地地址（一般为 `http://localhost:5173`）。请**同时**启动后端（`8080`），否则列表会报错或为空。

### 5.3 生产构建与预览

```bash
npm run build
npm run preview
```

生产环境未配置 `VITE_API_BASE` 时请求同源 `/api`（见 `src/utils/http.ts`）。部署时请配置环境变量或反代（见第 9 节）。

## 7. 接口说明

### 7.1 请求

| 项目 | 说明 |
|------|------|
| 开发环境路径 | `GET /api/v1/special/selectByTitle`（经代理转发到后端） |
| 查询参数 | `title`：标题关键词；`pageNum`：页码；`pageSize`：每页条数 |
| 编码 | `title` 为中文时 URL 编码由 Axios 自动处理 |

实现位置：`src/api/special.ts` 中的 `fetchSpecialByTitle`。

### 7.2 响应体兼容策略

后端返回结构可能因项目而异。前端对**常见分页包装**做了兼容，会从响应中**自动提取列表数组与总数**：

- 顶层或 `data` 下的字段：`list`、`records`、`rows`、`items`、`dataList`（**其一**为数组即可）。
- 总数字段：优先读取 `total`、`totalCount`，或在 `data` 嵌套对象中的同名字段。

若后端只返回列表、没有 `total`，前端会用**当前页列表长度**作为兜底总数（分页可能不完整；建议后端补全 `total`）。

若你的接口字段名不在上述列表中，请在 `src/api/special.ts` 中扩展 `pickRows` / `pickTotal` / `normalizeItem`。

## 8. 列表项字段映射（normalizeItem）

前端展示使用的统一结构为 `SpecialItem`（见 `src/types/special.ts`）。后端单条 DTO 与当前项目约定示例：

```json
{
  "id": "string",
  "title": "string",
  "banner": "string",
  "introduction": "string",
  "isFollowing": "string",
  "followersCount": 0,
  "viewCount": 0,
  "updated": 0
}
```

映射说明：

| 前端字段 | 含义 | 后端字段 / 说明 |
|----------|------|----------------|
| `cover` | 封面图 | 优先 `banner`、`bannerUrl`，其次历史兼容字段 `cover` 等 |
| `introduction` | 简介文案 | `introduction`，卡片内最多两行省略 |
| `isFollowing` | 是否已关注 | `isFollowing`（字符串 `"true"` / `"1"` 或布尔都会解析） |
| `followersLabel` | 关注人数展示 | `followersCount`（及兼容 `followers`、`followCount`） |
| `visitLabel` | 浏览量展示 | `viewCount`（及兼容 `views`、`readCount` 等） |
| `updateLabel` | 更新时间 | **优先**数值 `updated`：按 Unix 时间戳解析（&lt; 1e12 视为秒，否则毫秒），格式化为 `YYYY-MM-DD 更新`；无效时再回退 `updateTime`、`updatedAt` 等字符串字段 |
| `title` / `id` | 标题 / ID | `title`、`id`（兼容 `name`、`specialId` 等） |

浏览量、关注数会格式化为中文习惯（如过万显示为「x.x 万」）。

无封面时使用占位图服务；图片加载失败会回退到占位图。

## 9. 跨域与 baseURL

### 9.1 开发环境（Vite 代理）

`vite.config.ts`：

```ts
server: {
  proxy: {
    '/api': {
      target: 'http://localhost:8080',
      changeOrigin: true,
    },
  },
},
```

因此 HTTP `baseURL` 在开发模式下为 **`/api`**（见 `src/utils/http.ts`），请求路径与后端 `/api/v1/...` 一致。

### 9.2 生产环境建议

当前 `import.meta.env.DEV` 为 `false` 时，`baseURL` 写死为 `http://localhost:8080/api`，仅便于本地联调。**上线前**建议改为：

- `import.meta.env.VITE_API_BASE` 等环境变量；或  
- 与前端同源的相对路径 `/api`（由 Nginx 等反向代理到 Java 服务）。

并在根目录增加 `.env.production` 示例（需自行创建并加入 `.gitignore` 若含敏感信息）：

```env
VITE_API_BASE=https://your-domain.com/api
```

同时在 `client.ts` 中读取 `import.meta.env.VITE_API_BASE`（具体写法以你团队规范为准）。

## 11. 页面交互说明

- **搜索**：修改关键词后点击「搜索」或输入框回车；会重置到第 **1** 页并重新请求。
- **分页**：切换页码或每页条数会重新请求；改变每页条数时会回到第 1 页。
- **加载中**：使用 `el-skeleton` 占位。
- **失败**：Axios 异常时 `ElMessage.error` 提示，并清空列表（可改为展示重试按钮）。

「关注专题」按钮当前为**静态展示**，未接登录或关注接口；需要时可在此处调用你的业务 API。

## 12. 常见问题（FAQ）

**Q：浏览器控制台报 Network Error / 404？**  
A：确认后端已启动且路径为 `/api/v1/special/selectByTitle`；确认 Vite 代理 `target` 端口与后端一致。

**Q：列表有数据但总数不对？**  
A：检查后端是否返回 `total`（或文档第 7.2 节支持的字段名）；若无总数，前端会用当前页条数兜底。

**Q：标题、封面显示不对？**  
A：对照第 8 节字段映射，在 `normalizeItem` 中增加你的字段名。

**Q：打包后访问不了接口？**  
A：检查第 9.2 节 `baseURL` 与部署环境域名、HTTPS、反向代理是否配置正确。

**Q：能否减小 Element Plus 体积？**  
A：可改为按需引入组件与样式，并配合 `unplugin-vue-components`、`unplugin-auto-import`（需在文档外单独做一轮改造）。
